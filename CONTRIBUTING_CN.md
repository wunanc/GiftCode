# 参与贡献 (Contributing to GiftCode)

首先，非常感谢你考虑为 GiftCode 项目做出贡献！正是因为有像你一样的开发者，开源社区才能够不断进步和繁荣。

## 🐛 提交 Bug
如果你在插件中发现了漏洞或报错，请在 Issues 页面提交反馈。
为了让我们能更快地定位问题，请在反馈时提供以下信息：
- 服务器核心及版本（例如：Paper 1.20.1）
- GiftCode 的插件版本
- 问题的详细描述
- 如何复现这个 Bug（具体步骤）
- 完整的控制台报错日志（请使用 Pastebin 等平台，不要直接截图）

## 💡 提交新功能建议
我们非常欢迎新的想法！如果你有好的功能建议，请提交一个 Issue，并注明 `[Feature]` 标签。请详细说明该功能的作用，以及它能为服主或玩家带来什么便利。

## 💻 提交 Pull Request (PR)
我们非常乐意接受开发者提交的 PR（代码合并请求）来修复 Bug 或添加新功能。为了保证代码质量，请遵循以下流程：
1. **Fork** 本仓库到你的个人 GitHub 账号下，并克隆到本地。
2. 从 `master` 或 `main` 分支创建一个新的特性分支（例如：`git checkout -b feature/your-feature-name`）。
3. 提交你的代码更改（**Commit**），请确保提交信息简洁明了。
4. 在本地的 Minecraft 测试服务器中**测试**你的更改，确保没有引发新的错误。
5. 将代码 **Push** 到你的 Fork 仓库，并向我们发起 Pull Request。

### 👨‍💻 代码规范 (Code Guidelines)
- 请遵循标准的 Java 编码规范（如驼峰命名法、有意义的变量名等）。
- 确保你的代码在编译时不会产生大量警告（Warnings）。
- 若需要添加控制台输出，请使用本项目内置的 `XLogger` 工具类，并**优先使用 MiniMessage 语法**，而不是旧版的 `&` 颜色代码。
- 涉及复杂逻辑的地方，请添加适量的注释，以便其他开发者阅读。

感谢你对本项目的支持与贡献！
